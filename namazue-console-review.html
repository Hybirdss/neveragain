interaction logic

Navigation by focus, not by page.

The page itself should explain the console behavior: a single screen whose meaning changes as focus changes.

Event click

Event click should lock the quake, reframe the right rail, and clarify the impact field without changing pages.

  • Map focus tightens.
  • Snapshot changes from calm to live.
  • Check stack reorders.
Asset click

Asset click should shift context from quake-wide to asset-specific reasoning while preserving the main shell.

  • Right rail becomes current focus context.
  • Map highlights one asset.
  • Analyst note becomes more concrete.
Scenario shift

Scenario shift is the counterfactual layer. It should re-run consequences and explain the delta in one sentence.

  • Impact field changes.
  • Exposure ranks change.
  • Check These Now changes with it.

visual system

Deep navy, cold metal, controlled escalation.

This page should reuse only the operator-console-compatible pieces of the old design system. The rest should be rebuilt around severity, precision, and low-noise surfaces.

Surface rules
  • Low-gloss plates, never glass-card overload.
  • Thin borders and measured blur only where it helps hierarchy.
  • The canonical console is always the dominant visual object.
Type and motion
  • Display type should feel industrial, not editorial.
  • Mono belongs on state, timing, and severity labels.
  • Motion is sparse: section reveal, state glow shift, scenario delta cue.

voice system

Trusted analyst, not news anchor, not chatbot.

The language of the page must prove that the product is calm, precise, and operationally serious.

Approved operator language
  • Operational impact elevated across coastal Tokyo.
  • Three assets require immediate inspection.
  • Hospital access posture changed under scenario shift.
  • Why this changed: shaking moved east toward the bay corridor.
Embedded analyst framing

AI should exist as Analyst Note, Reasoning, Why this changed, or Suggested checks. It should never behave like a chat-first assistant.
