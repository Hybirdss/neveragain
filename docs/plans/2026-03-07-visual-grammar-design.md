# Visual Grammar Design

Status: approved direction snapshot  
Date: 2026-03-07

## Design Rule

The console must be visually arresting without becoming visually noisy.

The governing rule is:

`hazard is animated, consequence is asserted`

This means:

- earthquakes, waves, and intensity spreads move
- operational consequences appear crisp, deliberate, and ranked
- infrastructure is never allowed to shimmer like a game HUD

## Hierarchy

The screen has exactly three visual layers.

### 1. Background Layer

Purpose:

- orientation
- coastlines
- region and city context
- quiet sense of Japan as a living system

Rules:

- low contrast
- almost no motion
- never competes with event or consequence layers

### 2. Hazard Layer

Purpose:

- show how the physical event propagates

Rules:

- event dot is the ignition point
- P-wave is thin, precise, early, cold
- S-wave is thicker, slower, heavier, warmer
- intensity spreads outward like ink in water

This is the product’s signature visual.

### 3. Consequence Layer

Purpose:

- show what needs attention

Rules:

- selective emphasis
- strong state colors only when consequence is meaningful
- labels appear later than hazard, not at the same instant
- high-confidence operator checks dominate over decorative detail

## Event Choreography

The approved sequence remains:

1. epicenter appears
2. P-wave expands
3. S-wave follows
4. intensity field spreads
5. infrastructure consequence asserts
6. priority queue populates

The sequence must feel causal, not simultaneous.

## Bundle-Specific Visual Language

Each bundle needs its own motion and emphasis grammar.

### Seismic

- strongest motion
- clearest propagation
- truth and confidence surfaces

### Maritime

- motion comes from vessel headings and trails
- red is reserved for actual consequence, not routine traffic
- hazardous and passenger traffic earn distinct emphasis

### Lifelines

- rail should read as corridor stress, not generic colored lines
- power should read as network posture, not isolated dots
- nuclear should read as controlled, high-consequence vigilance

### Medical

- hospital symbols should communicate readiness and access stress
- the visual mood should be urgent but not alarmist

### Built Environment

- city-scale expression only
- avoid visual clutter at national zoom
- color change should feel like consequence bloom, not glitter

## Calm Mode

Calm mode must still feel alive.

Allowed:

- slow vessel movement
- subdued rail presence
- low-noise earthquake field
- quiet system telemetry

Forbidden:

- empty black map
- exaggerated idle animations
- rotating world or decorative drift

## Event Mode

Event mode should sharply increase meaning, not just brightness.

What changes:

- hazard propagates
- affected infrastructure asserts
- panel language becomes decisive
- operator queue becomes the right-rail climax

What does not change:

- background remains restrained
- controls remain secondary
- random pulses and glow storms remain forbidden

## Readability Guardrails

- controls must never overpower truth
- bundle chips must never read louder than the event
- legends and secondary metrics must stay subordinate
- motion must stop once it no longer adds meaning
- red must remain scarce enough to matter

## Visual Success Test

If an operator glances once, they should be able to tell:

- where the event is
- how it is spreading
- which domains are shifting posture
- what requires immediate inspection

If the answer is only “this looks cool,” the design failed.
