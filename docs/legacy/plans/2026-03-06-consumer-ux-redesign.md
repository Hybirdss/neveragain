# Consumer UX Redesign Plan

> **Goal**: "분석 도구" 느낌 → "매일 들어와서 보는 서비스"로 전환
> **Core Insight**: 일반인은 "지진이 있었다"를 알고 싶은 게 아니라, "나는 괜찮은가?"를 알고 싶다.

---

## 1. 현재 문제 진단

### 왜 "분석 도구" 느낌이 나는가?

| 증상 | 원인 | 영향 |
|------|------|------|
| 첫 화면에서 글로브 + 점만 보임 | 0-click value 없음 | 유레타씨가 3초 안에 이탈 |
| 좌측 패널이 데이터 테이블처럼 보임 | 정보 계층 없이 나열 | 전문가만 읽을 수 있음 |
| AI 분석이 3-탭에 숨겨져 있음 | Layer 1(한 줄)이 묻힘 | 서비스의 핵심 차별화가 사용자에게 전달 안 됨 |
| 규모/깊이/좌표가 동일 크기 | 모든 정보가 동등한 비중 | 시선이 분산되어 핵심 파악 불가 |
| 쓰나미 정보가 작은 배지 | 생명 관련 정보가 눈에 안 띔 | Safety-first 원칙 위반 |
| 온보딩/가이드 없음 | 기능 발견성 0 | 처음 온 사람이 뭘 해야 할지 모름 |
| 모바일 peek 시트에 규모 숫자만 | AI 한 줄 해석이 안 보임 | 모바일 사용자(80%)에게 가치 전달 실패 |

### 핵심 질문: 유레타씨가 뭘 보고 싶은가?

```
"방금 흔들렸어" → 앱을 연다 →

현재:  [글로브] + [M4.2 | 34.5°N 135.2°E | 12km]  ← "그래서 뭐?"
이상:  [글로브] + [오사카에서 진도4. 선반 위 물건에 주의. 쓰나미 걱정 없음.]  ← "아 그렇구나"
```

## 2. 설계 원칙

### P1: Meaning First, Numbers Second
- AI 한 줄 해석이 가장 먼저, 가장 크게
- 숫자(규모/깊이/좌표)는 보조 정보로 작게

### P2: Progressive Disclosure
- Layer 1 → Layer 2 → Layer 3로 자연스러운 깊이 전환
- 일반인은 Layer 1에서 만족, 전문가는 Layer 3까지 탐색

### P3: Emotional Color Language
- 색상만으로 "괜찮다/조심해/위험해"를 전달
- 전문 용어 대신 색상 + 아이콘 + 평문

### P4: Mobile-First, Globe-Background
- 모바일 사용자가 80%. 바텀시트가 주 인터페이스
- 글로브는 몰입감을 주는 배경이지 분석 도구가 아님

### P5: Safety Breaks Through
- 쓰나미 경보는 모든 UI를 뚫고 최상단에 표시
- 숨길 수 없음. dismiss 불가.

### Before (현재)
```

├── [LIVE] 모드 레이블
├── 검색 (Cmd+K)

    ├── MMI 막대 (14px, 거의 안 보임)
    └── Analysis 탭
```

### After (제안)
```

│   ├── 쓰나미 걱정 없음 ✓                       ← Safety 즉시 확인 (green)
│   ├── M4.2 · 12km · 3분 전                     ← 숫자는 작게 (11px, 회색)
│   └── [자세히 보기 →]
│
├── [Recent List] ← 간소화된 리스트
│   ├── ● M3.1 도쿄 · 7분 전 · "약한 흔들림"
│   ├── ● M2.8 치바 · 15분 전
│   └── ...
│
└── 선택 시 → Slide-over Detail
    ├── [AI 해설 카드] ← Layer 1/2 통합
    │   ├── "왜 이 지진이 발생했나?" (3-5문장)

    │   └── 행동 지침 (구체적)
    ├── [진도 시각화] ← 새 디자인
    │   └── 수평 바 → 큰 숫자 + 의미 텍스트
    ├── [전문가 분석 ▼] ← 접힘 (Expandable)
    └── [데이터 ▼] ← 접힘
```

## 4. 컴포넌트 상세 설계

### 4.1 Hero Card (신규)

**파일**: `ui/heroCard.ts`

서비스의 첫인상을 결정하는 가장 중요한 컴포넌트.
0-click으로 "지금 뭐가 중요한지"를 전달.

```
┌─────────────────────────────────────┐
│  [JMA 색상 세로 바]                  │
│                                     │
│  오사카에서 진도 4               ← AI headline (18px, bold)
│  棚の物が落ちるかもしれません      ← AI one_liner (15px)
│                                     │
│  ✓ 津波の心配はありません          ← Tsunami status (green/red)
│                                     │
│  M4.2 · 深さ12km · 3分前           ← Meta (11px, tertiary)
│                                     │
│  [詳しく見る →]                     ← CTA
└─────────────────────────────────────┘
```

**상태별 분기**:
- 분석 있음: AI headline + one_liner
- 분석 로딩 중: skeleton + "分析を準備中..."
- 분석 없음 (M<4): 규모/위치만 표시, AI 없음
- 지진 없음 (24h): "最近24時間に大きな地震はありません" (안심 메시지)

**색상 규칙**:
- 좌측 세로 바 색상 = JMA 계산 결과 색상
- 배경 tint = 같은 색상의 매우 낮은 opacity (5%)
- 쓰나미 없음 = green, 있음 = red pulsing

**모바일 대응**:
- 바텀시트 peek 상태에서 Hero Card가 보임
- 글로브 위에 떠있는 유일한 정보 → 시선 집중

**변경점**:

- 여진 클러스터링 유지하되 "＋3件の余震" 배지로 간소화

- 행 높이 증가: 48px → 56px (터치 타깃)

### 4.3 Detail Panel 재구성

**파일**: `ui/detailPanel.ts` (리팩토링)

현재: 규모/깊이/좌표가 메인 → Expert 중심
제안: AI 해설이 메인 → Consumer 중심

```
┌─────────────────────────────────────┐
│ ✕                                   │
│                                     │
│ M6.8 能登半島沖                      │ ← 규모 + 장소 (heading)
│ 2026年3月6日 14:32 · 深さ12km        │ ← 시간/깊이 (subtext)
│                                     │
│ ┌─ 진도 시각화 ──────────────────┐   │
│ │  震度 5強                       │   │ ← 큰 숫자 (32px)
│ │  "立っていることが困難"          │   │ ← JMA 체감 설명
│ │  [0][1][2][3][4][5-][5+][6-]... │   │ ← 강도 바 (24px, 터치 가능)
│ └────────────────────────────────┘   │
│                                     │
│ ┌─ 쓰나미 판정 ──────────────────┐   │
│ │ ⚠ 津波に注意してください        │   │ ← moderate = yellow bar
│ │ M6.8, 浅い海底, プレート境界型   │   │ ← 근거 표시
│ └────────────────────────────────┘   │
│                                     │
│ ▼ この地震について (AI 해설)          │ ← Expandable Section
│   "太平洋プレートが..."              │
│   "今後24時間以内に..."              │
│   [推奨行動]                         │
│   [よくある質問]                     │
│                                     │
│ ▼ 専門家向け分析                     │ ← Expandable Section
│ ▼ データ                             │ ← Expandable Section
│                                     │
│ ─────────────────────────────────── │
│ ⓘ 参考情報です。公式情報は気象庁へ    │ ← Disclaimer (항상 표시)
└─────────────────────────────────────┘
```

**핵심 변경**:
1. 3-탭 → Expandable Sections (아코디언)
   - 탭은 "선택해야 한다"는 인지 부하 발생
   - 아코디언은 스크롤로 자연스럽게 탐색
   - "이 지진에 대해" 섹션이 기본 열림 (Layer 1+2)
   - "전문가 분석" / "데이터"는 기본 닫힘 (Layer 3)

2. 진도 시각화 강화
   - 14px MMI 바 → 32px 큰 숫자 + 체감 설명 텍스트
   - "震度5強: 立っていることが困難" ← 일반인이 바로 이해

3. 쓰나미 판정을 독립 섹션으로 승격
   - 배지 → 카드 (배경색 + 좌측 바 + 근거 텍스트)
   - "none"이어도 "쓰나미 걱정 없음 ✓"으로 명시적 표시

4. Disclaimer 위치 변경
   - 콘텐츠 아래에 묻힘 → 패널 하단 고정 (sticky)
   - 항상 보이되, 시각적으로 겸손하게

## 5. 시각 디자인 변경

### 5.1 타이포그래피 계층 강화

현재 문제: 제목과 본문의 시각적 차이가 작음 (13px vs 11px)

```css
/* Before */
--text-base: 13px;
--text-sm: 11px;
/* 차이: 2px → 거의 구분 안 됨 */

/* After — 더 명확한 계층 */
--text-hero:    24px;  /* Hero card headline */
--text-heading: 18px;  /* Section titles */
--text-body:    15px;  /* 본문 (기존 13px → 15px) */
--text-meta:    12px;  /* 보조 정보 */
--text-caption: 10px;  /* 캡션/라벨 */

/* Weight도 계층화 */
--weight-bold:    700;  /* Hero headline */
--weight-medium:  500;  /* Section titles, 강조 */
--weight-regular: 400;  /* 본문 */
```

### 5.2 색상: "의미 색상 시스템" 통합

현재: JMA 10단계 색상 + Safety 4단계 색상이 별개
제안: 통합 시맨틱 색상 시스템

```css
/* Severity scale — 모든 곳에서 동일 */
--severity-safe:     #4ade80;  /* 괜찮다 */
--severity-info:     #60a5fa;  /* 참고 */

/* 용도별 매핑 */
JMA 0-2  → safe (파랑/초록)
JMA 3-4  → caution (노랑)
JMA 5-   → warning (주황)
JMA 5+   → danger (빨강)
JMA 6+,7 → critical (진한 빨강)

Tsunami none     → safe
Tsunami low      → info
Tsunami moderate → warning
Tsunami high     → critical
```

## 6. 인터랙션 흐름 재설계

### Flow A: 유레타씨 — "방금 흔들렸어" (Mobile)

```
1. 앱을 연다 (혹은 PWA 알림 탭)
2. 글로브가 이미 지진 위치로 줌 (자동 카메라)
3. 바텀시트 peek:
   "오사카에서 진도4. 선반 위 물건에 주의. 쓰나미 걱정 없음."
4. 스와이프 업 (half):
   AI 해설 카드 + 최근 지진 리스트
5. 더 스와이프 (full):
   여진 전망 + 행동 지침 + FAQ
6. 만족 → 앱 닫기 (30초 내에 완료)
```

**핵심**: Step 3에서 이미 가치 전달 완료. 99%의 유레타씨는 여기서 끝.

### Flow B: 지진통씨 — "오늘 뭐 있었지?" (Desktop)

```
1. 매일 접속
2. 좌측 패널 Hero Card:
   가장 큰 최근 지진의 AI 요약
3. 리스트 스크롤: 24h 지진 확인
4. 관심 지진 클릭 →
   Detail slide-over: AI 해설 + 등진도선 오버레이
5. "전문가 분석" 섹션 열기 → 깊은 분석
6. 타임라인으로 과거 범위 변경
```

### Flow C: 보사이씨 — "리포트 필요" (Desktop)

```
1. Cmd+K → "渋谷区 지진 이력"
2. 카메라 이동 + 주변 지진 마커 표시
3. AI 지역 리포트 자동 생성
4. "데이터" 섹션 → JSON 다운로드
5. 프레젠테이션 모드로 화면 캡처
```
